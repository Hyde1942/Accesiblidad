Qué es la Accesibilidad Web?

La Accesibilidad Web es simplmente garantizar que un sitio o servicio web este disponible para el mayor número de personas.

Imagenes

Tienen que tener un alt tag descriptivo pero no demasiado largo. En el caso de tener que suministrar una descripción mas detallada utilizar otro metodo, como ARIA para suministrar la descripción.


									<img src="images/path.jpg" alt="Descripcion 125ch max" />

Para las images decorativas se tiene que poner un alt tag vacio.


									<img src="images/path.jpg" alt="" role="presentation" />

En el caso de imagenes mas complejas, como Infografics, diagramas de flujo, etc. Suministrar una descripcion mas detallada usando:

longdesc Examples

Contraste y Color

"Contraste" son colores en segmentos opuestos de la rueda del color.

Para cumplir la normativa AA, El contraste minímo de un texto normal debe ser 4:5:1

Para cumplir la normativa AAA, El contraste minímo de un texto debe ser 7:1

Font-size & Line Height

El font-size del texto del página debería setearse en un valor entre los 15px - 20px. De acuerdo con la W3C el line-height debería ser 1.5 veces el tamaño del line spacing. También el largo de los textos de no debería de exceder los 85 carácteres. 

								
								.p {
									font-size: 1em;
									max-width: 65ch; /* Limits paragraph width */
									line-height: 1.5;
								}

JavaScript & ARIA

No ARIA is better than Bad ARIA

Accessible Rich Internet Applications (WAI-ARIA 1.1). Es una serie de propiedades que se añadieron a la especificación de HTML, para hacer las aplicaciones y el contendido web (JavaScript) más accesible.

ARIA se compone de tres grandes pilares

  • Role
  • States
  • Properties

Role

Role, describe una serie de elementos abstractos, widgets, landmarks, live regions, etc. Que no forman parte de la especificación de HTML 4. Los elementos abstractos no sé deben usar. Y los demás hay que entender ARIA para poder usuarlos.

Live Regions

Nos permiten anunciar algo sin necesidad que el usuario intervenga. Esto es muy útil a la hora de manejar updates de una API, AJAX (XHR), validaciones.

  • Assertive: Se interrumpe cualquier acción que el usuario este realizando y se anuncia lo que se incluya en el live region
  • Polite: Se pone el anuncio en un queue. Esperando que se termine de realizar la accion que el usuario esta realizando; Y después se realiza el anuncio.

Code Examples

Es importante que el live Region se encuentre en el DOM al momento del load de la página. Ya que el screen reader tiene que registrar y mantener track del container que sirve como Live Region.

							
								<div aria-live="polite"></div>
